# Примеры использования Agent Doctor

## Базовое использование

### Запуск диагностики

Просто скажите агенту:

```
продиагностируй себя
```

или

```
самодиагностика
```

или

```
health check
```

---

## Сценарий 1: После обновления OpenClaw

**Ситуация:** Вы обновили OpenClaw через `npm update -g openclaw`

**Что делать:**

1. Запустить диагностику:
```
продиагностируй себя
```

2. Агент проверит все системы

3. Типичные проблемы после обновления:
   - memory-core плагин отключен
   - WAL mode сброшен на delete
   - Конфиг требует обновления

4. Агент предложит фиксы, подтвердите их

## Сценарий 2: Агент "забывает" разговоры

**Симптомы:**
- "Не помню что мы обсуждали 10 минут назад"
- memory_search не находит недавние записи
- Повторяет одни и те же вопросы

**Диагностика:**

```
самодиагностика
```

**Вероятные причины:**
1. ❌ WAL mode = delete (самая частая причина!)
2. ❌ memorySearch отключен
3. ❌ Embeddings не работают

**Решение:**

Агент найдет проблему и предложит:
```bash
sqlite3 ~/.openclaw/memory/main.sqlite "PRAGMA journal_mode=WAL;"
```

Подтвердите и перезапустите gateway.

## Сценарий 3: Кроны не работают

**Симптомы:**
- Крон в списке, но не запускается
- failureCount > 0
- Ошибки в логах

**Диагностика:**

```
проверь систему
```

**Блок "Кроны" покажет:**
- Какие кроны отключены
- Когда был последний запуск
- Есть ли ошибки

**Типичные проблемы:**
1. Модель не указана полным путем (нужно `anthropic/claude-sonnet-4-6`)
2. API ключ не настроен
3. Крон отключен (enabled: false)

**Пример фикса:**

```bash
openclaw cron edit watchdog-v2 --model "anthropic/claude-sonnet-4-6"
openclaw cron enable watchdog-v2
```

## Сценарий 4: Gateway не запускается

**Симптомы:**
- `openclaw status` → "not running"
- Ошибки при старте

**Диагностика:**

```
что с тобой не так
```

**Блоки "Gateway" и "Система" покажут:**
- Версия Node.js (должна быть >= 20)
- Занят ли порт
- Ошибки в логах

**Типичные решения:**

1. Порт занят:
```bash
lsof -ti:3000 | xargs kill -9
openclaw gateway start
```

2. Старый Node.js:
```bash
brew upgrade node  # macOS
nvm install 20     # Linux
```

3. Битый конфиг:
```bash
jq empty ~/.openclaw/openclaw.json
# Если ошибка - исправить вручную
```

## Сценарий 5: Мало места на диске

**Симптомы:**
- Ошибки записи
- База данных не растет
- Кроны падают

**Диагностика:**

```
self-check
```

**Блок "Система" покажет:**
- Свободное место (должно быть >1GB)

**Решение - очистка:**

```bash
# Старые логи
rm -rf ~/.openclaw/logs/*.log.old

# Кеш
rm -rf ~/.openclaw/cache/*

# Бэкапы старше 30 дней
find ~/.openclaw/backups -mtime +30 -delete
```

Агент предложит эти команды автоматически.

## Сценарий 6: Gateway доступен из интернета

**Опасность:** Если bind = 0.0.0.0, любой может подключиться к вашему агенту!

**Диагностика:**

```
проверь безопасность
```

**Блок "Безопасность" покажет:**
- Gateway bind: 0.0.0.0 ❌ (опасно!)

**Решение:**

```bash
jq '.gateway.bind = "127.0.0.1"' ~/.openclaw/openclaw.json > /tmp/config.json && mv /tmp/config.json ~/.openclaw/openclaw.json
```

Перезапустить gateway:
```bash
launchctl kickstart -k gui/$(id -u)/com.openclaw.gateway
```

## Сценарий 7: Автоматическая диагностика (крон)

**Задача:** Проверять систему каждый день автоматически

**Решение - создать крон:**

```bash
openclaw cron add daily-health-check \
  --schedule "0 8 * * *" \
  --model "anthropic/claude-sonnet-4-6" \
  --isolated \
  --payload '{
    "kind": "agentTurn",
    "message": "Запусти диагностику agent-doctor. Если найдешь критичные проблемы (WAL mode, memory-core, bind=0.0.0.0) - отправь алерт через message tool. Если все ОК - молчи."
  }'
```

Теперь каждое утро в 8:00 агент проверит себя и сообщит только о проблемах.

## Сценарий 8: Bash-скрипт диагностики

**Задача:** Быстрая проверка без агента (для CI/CD или скриптов)

**Решение:**

```bash
bash skills/agent-doctor/auto-diagnostic.sh
```

**Вывод:**
```
🏥 AGENT DOCTOR - Автодиагностика
================================

🧠 ПАМЯТЬ
--------
Проверка: SQLite база существует... ✅ OK
Проверка: WAL mode включен... ✅ OK
...

💾 СИСТЕМА
---------
Проверка: Node.js версия >= 20... ✅ OK
...

================================
✅ Все проверки пройдены!
```

Exit code:
- 0 = все ОК
- 1 = найдены проблемы

Можно использовать в скриптах:
```bash
if bash skills/agent-doctor/auto-diagnostic.sh; then
    echo "Система здорова"
else
    echo "Нужна диагностика"
    # Уведомить админа
fi
```

## Сценарий 9: Выборочное применение фиксов

**Ситуация:** Диагностика нашла 5 проблем, но вы хотите исправить только некоторые

**Пример:**

```
Агент: Найдено 5 проблем. Исправить? (да/нет)

Вы: 1, 3, 5

Агент: Применяю фиксы #1, #3, #5
```

Или:

```
Вы: все кроме 2

Агент: Применяю фиксы #1, #3, #4, #5
```

## Сценарий 10: Диагностика перед важной работой

**Задача:** Убедиться что все работает перед стримом/презентацией

**Чеклист:**

```
1. Запустить: продиагностируй себя
2. Проверить все блоки = ✅
3. Если ⚠️ или ❌ - исправить ДО начала
4. Убедиться:
   - WAL mode = wal
   - memorySearch = true
   - Gateway работает
   - Кроны без ошибок
   - Диск >1GB
```

**Если все ✅** - можно работать!

## Частые вопросы

**Q: Как часто запускать диагностику?**

A: 
- После каждого обновления OpenClaw (обязательно!)
- Раз в неделю для профилактики
- Перед важной работой
- При странном поведении агента

**Q: Можно ли автоматизировать фиксы?**

A: Нет! Все фиксы требуют вашего подтверждения. Это безопасность.

**Q: Что делать если диагностика сама падает?**

A: 
1. Проверить: `openclaw status`
2. Посмотреть логи: `tail -50 ~/.openclaw/logs/gateway.log`
3. Запустить bash-скрипт: `bash skills/agent-doctor/auto-diagnostic.sh`

**Q: Какие проблемы самые критичные?**

A:
1. ❌ WAL mode = delete (агент слепой!)
2. ❌ memory-core отключен (память мертва)
3. ❌ bind = 0.0.0.0 (дыра в безопасности)

Это ВСЕГДА чинить первым делом!

## Шпаргалка команд

```bash
# Диагностика
openclaw status
openclaw cron list
jq '.memorySearch' ~/.openclaw/openclaw.json

# WAL mode
sqlite3 ~/.openclaw/memory/main.sqlite "PRAGMA journal_mode;"
sqlite3 ~/.openclaw/memory/main.sqlite "PRAGMA journal_mode=WAL;"

# Перезапуск gateway (безопасно)
launchctl kickstart -k gui/$(id -u)/com.openclaw.gateway

# Очистка
rm -rf ~/.openclaw/logs/*.log.old
rm -rf ~/.openclaw/cache/*

# Проверка базы
sqlite3 ~/.openclaw/memory/main.sqlite "SELECT source, COUNT(*) FROM memory_chunks GROUP BY source;"
```
